# 위클리 — 팀 일정 조율 프로토타입

같은 회사 동료 6명이 다음 주 안에 1시간 회의를 잡는 경험을 설계한 데스크톱 웹 프로토타입입니다.

이 제품은 빈 시간을 보여주는 캘린더가 아니라, 참석자 우선순위(필수/선택/공유)와 숨은 제약(점심 직후 비선호, 외근 가능성, 회의실)을 함께 확인해 **"이 시간으로 회의를 잡아도 되는지"를 판단하도록 돕는** 일정 조율 도구입니다.

## 실행 방법

```bash
npm install
npm run dev
```

브라우저에서 `http://localhost:5173` 접속. 1440px 데스크톱 기준으로 설계되었습니다.

## 체험 흐름

오른쪽 회의 조율 패널의 버튼을 따라가면 전체 흐름을 처음부터 끝까지 볼 수 있습니다.

1. 회의 만들기 — 회의 길이·기간을 정하고 참석자를 검색해 선택 (선택 전에는 내 캘린더만 보임)
2. 회의 조건 확인 → 꼭 참석해야 할 사람 정하기 (필수/선택/공유 그룹으로 역할 판단)
3. 후보 시간 3개를 확정 가능도 기준으로 비교 → 추천 이유 확인 → 제안
4. 응답 대기 중 참여자(정하늘) 화면으로 전환 — 드래그로 안 되는 시간/피하고 싶은 시간 표시, 원탭 대안 선택
5. 참석자 응답 확인 → 회의 확정
6. 확정 후 필수 참석자 일정 변경 알림 도착 → 기존 시간 유지 가능 여부 확인
7. 기존 응답 + 최신 캘린더 상태를 다시 확인해 대체 시간 제안
8. 변경 제안 응답 확인 → 변경 확정

후보 카드를 클릭하면 중앙 캘린더에서 해당 시간이 강조됩니다. 다른 사람 일정은
실제 캘린더 제품처럼 내용 없이 '바쁨'으로만 표시되고, 연차·외근 같은 부재 상태만 공개됩니다.

## 기술 스택

- React 18 + TypeScript + Vite
- Tailwind CSS v4
- 외부 서버·로그인·캘린더 연동 없음 (모든 데이터는 `src/data/mockData.ts`)

## 구조

```
src/
  App.tsx                  # 단계(step) 상태 관리
  types.ts
  data/mockData.ts         # 회의 정보, 참석자 6명, 후보/대체 시간, 팀 일정
  components/
    Header.tsx             # 상단 헤더 (주간 내비게이션, 참석자 아바타)
    Sidebar.tsx            # 왼쪽 사이드바 (메뉴, 참석자 목록)
    CalendarGrid.tsx       # 중앙 주간 캘린더 (단계별 후보/제안/확정/변경 블록)
    MeetingPanel.tsx       # 오른쪽 회의 조율 패널 (11단계 플로우)
    AttendeeCard.tsx       # 참석자 카드 + 필수/선택/공유 segmented control
    CandidateCard.tsx      # 후보 시간 카드 (확정 가능도, 판단 근거)
    StatusBadge.tsx        # 상태 배지 (조건 확인 / 제안 중 / 확정 완료 / …)
    ConfidenceBadge.tsx    # 확정 가능도 배지
```
