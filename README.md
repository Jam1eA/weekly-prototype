# 위클리 — 팀 일정 조율 프로토타입

같은 회사 동료 6명이 다음 주 안에 1시간 회의를 잡는 경험을 설계한 데스크톱 웹 프로토타입입니다.

기존 캘린더는 일정 충돌은 보여주지만, 참석자의 중요도(필수/선택)·비선호 시간·아직 확정되지 않은 외근 같은 제약까지 함께 판단해주지 못합니다. 이 제품은 후보 시간을 **"바로 잡아도 되는 시간 / 특정 참석자에게만 확인이 필요한 시간 / 다른 시간이 나은 경우"로 구분**해서 — 문제가 없으면 바로 잡고, 불확실한 사람에게만 확인하고, 변경이 생기면 영향받는 사람만 다시 확인하는 일정 조율 도구입니다.

## 실행 방법

```bash
npm install
npm run dev
```

브라우저에서 `http://localhost:5173` 접속. 1440px 데스크톱 기준으로 설계되었습니다.

## 체험 흐름 (권장 경로, 약 3분)

오른쪽 회의 조율 패널의 버튼을 따라가면 전체 흐름을 볼 수 있습니다. 데스크톱 브라우저 권장.

1. **회의 만들기** — 회의 정보와 참석자를 설정하는 동안 실시간 후보가 캘린더와 패널에 바로 나타난다
2. **후보 시간** — 후보 3개가 `바로 잡을 수 있어요 / 확인이 필요해요 / 다른 시간이 나아요`로 구분되고, 카드를 고르면 같은 화면에서 참석자 상태와 다음 행동이 바뀐다
3. **화요일 14:00 (바로 잡을 수 있어요)** — 걸리는 사람이 없으니 확인 없이 **이 시간으로 회의 잡기** → 즉시 확정
4. **확정 후 변경 알림 도착** — 박서준님 외근이 화요일로 확정되며 회의와 충돌
5. **영향받는 1명에게만 다시 확인** — **"박서준님 화면에서 답하기"로 참여자 응답 화면을 직접 체험** → 수요일 15:00로 변경 확정 (나머지 4명은 변경 알림만, 전원 재조율 없음)

곁가지 경로:
- **목요일 11:00 (확인이 필요해요)** — 박서준님 외근 가능성만 불확실 → "박서준님께 물어보기"로 그 사람에게만 짧게 확인, 응답이 오면 주최자가 최종 확정
- **수요일 13:00 (다른 시간이 나아요)** — 김민준님(필수) 일정과 충돌 → 확정·확인 대신 다른 후보 권장
- 재확인에서 박서준이 "이 시간은 어려워요"를 선택하면 → 다른 시간을 다시 찾는 복구 경로

다른 사람 일정은 실제 캘린더 제품처럼 내용 없이 '바쁨'으로만 표시되고, 연차·외근 같은 부재 상태만 공개됩니다.

## 참고한 서비스와 가져온 사용성

Google Calendar, Outlook Calendar, 되는시간 3개 서비스를 직접 사용하며 분석했고,
각 서비스에서 검증된 패턴을 아래와 같이 가져와 맥락에 맞게 다듬었습니다.

| 출처 | 가져온 패턴 | 적용된 화면 |
|---|---|---|
| Google Calendar | 회의 생성 → 참석자 추가 → 시간 찾기 순서 | 새 회의 만들기 → 조건 확인 플로우 |
| Google Calendar | 참석자 '선택사항으로 표시' 원클릭 토글 | 참석자 역할 배지 (기본 필수, 클릭으로 전환) |
| Google Calendar | 초대 이메일 발송 확인 모달 | 제안 보내기 확인 모달 |
| Outlook | 필수/선택 초대 대상 구분 | 필수·선택·공유 참석 구성 |
| Outlook | 선택한 시간에 몇 명이 참석 불가한지 즉시 표시 | 후보 블록·카드의 가용 인원 표시 |
| Outlook | 수락/미응답 그룹핑된 응답 상태 | 응답 대기 화면의 참석자별 상태 목록 |
| When2meet | 드래그로 가능 시간 칠하기 | 참여자 조건 그리드 (드래그 입력) |
| 되는시간 | 링크형 참여자 응답 화면, 캘린더 일정 자동 반영 | 참여자 응답 화면 (일정 프리필) |
| 되는시간 | 예약 완료 후 변경/취소 링크 | 응답 완료 화면의 '응답 수정하기' |

**의도적으로 따르지 않은 것** — 되는시간의 예약형 구조(주최자가 열어둔 시간 중 택 1)는
다자간 조율에 맞지 않아 제외했고, Outlook의 '미정' 응답은 확정 판단을 흐려 제외했습니다.
가능/불가 2값뿐인 When2meet의 한계는 '안 돼요/피하고 싶어요' 2단계로 확장했습니다.

## 기술 스택

- React 18 + TypeScript + Vite
- Tailwind CSS v4
- 외부 서버·로그인·캘린더 연동 없음 (모든 데이터는 `src/data/mockData.ts`)

## 구조

```
src/
  App.tsx                  # 단계(step) 상태 관리
  types.ts
  data/mockData.ts         # 회의 정보, 참석자 6명, 후보/대체 시간, 팀 일정
  components/
    Header.tsx             # 상단 헤더 (주간 내비게이션, 참석자 아바타)
    Sidebar.tsx            # 왼쪽 사이드바 (메뉴, 참석자 목록)
    CalendarGrid.tsx       # 중앙 주간 캘린더 (단계별 후보/제안/확정/변경 블록)
    MeetingPanel.tsx       # 오른쪽 회의 조율 패널 (단계별 플로우)
    AttendeeCard.tsx       # 참석자 카드 + 필수/선택/공유 segmented control
    CandidateCard.tsx      # 후보 시간 카드 (3분류 배지, 판단 근거)
    StatusBadge.tsx        # 상태 배지 (시간 고르는 중 / 확인 요청 중 / 확정 완료 / …)
    ConfidenceBadge.tsx    # 후보 3분류 배지 (바로 잡기 / 확인 필요 / 다른 시간)
```
